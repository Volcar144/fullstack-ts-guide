# Choosing a Framework

When you start learning web development, you quickly hear words like "React", "Next.js", "Vue", "SvelteKit", and "Angular". It can feel overwhelming. Let's break it down.

## What is a framework?

A **framework** is a collection of pre-built tools and conventions that help you build apps faster. Instead of solving every problem from scratch, a framework gives you a structured way to work.

Think of it like this:
- Building without a framework = cooking from raw ingredients
- Building with a framework = using a recipe and pre-made sauces

## The landscape in 2025

| Framework | Language | Best For |
|-----------|----------|----------|
| **Next.js** | TypeScript/JS | Full-stack apps, production sites |
| **Remix** | TypeScript/JS | Full-stack, great data loading |
| **SvelteKit** | Svelte | Smaller bundle, great DX |
| **Nuxt** | Vue | Vue ecosystem |
| **Angular** | TypeScript | Large enterprise apps |

## Why we're using Next.js

For a beginner aiming at full-stack TypeScript development, **Next.js** is the best choice right now because:

1. **Huge community** — millions of developers, tons of tutorials and Stack Overflow answers
2. **Full-stack in one codebase** — your frontend and backend (API) live together
3. **React-based** — React is the most popular UI library in the world; skills transfer everywhere
4. **Vercel deployment** — go from code to live URL in minutes
5. **TypeScript first** — it's designed with TypeScript in mind

## The learning stack we'll follow

```
HTML & CSS → JavaScript → TypeScript → React → Next.js → Full App
```

This is the most logical path. Each layer builds on the one before it. Let's start at the bottom.

<Callout type="tip">
  Don't worry about which framework is "the best". Learn the fundamentals well and you'll be able to pick up any framework later. The concepts transfer.
</Callout>
