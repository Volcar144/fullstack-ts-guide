# Introduction to React

React is a JavaScript library for building user interfaces. Created by Meta in 2013, it's now the most widely used UI library in the world.

## The problem React solves

Before React, building dynamic UIs meant manually manipulating the DOM:

```js
// Vanilla JS — tedious and error-prone
const count = 0;
const button = document.getElementById('increment');
const display = document.getElementById('count');

button.addEventListener('click', () => {
  count++;
  display.textContent = count; // manually update every time
});
```

As apps grew, this became unmaintainable. React introduced a better model:

```jsx
// React — describe what you want, React updates the DOM
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}
```

## The key ideas

### 1. Components
Your UI is broken into small, reusable **components**. A button is a component. A card is a component. A page is a component made of other components.

### 2. Declarative UI
You describe **what** the UI should look like based on the current state — not **how** to update it. React figures out the DOM updates.

### 3. One-way data flow
Data flows down from parent to child components via **props**. This makes data flow predictable and debugging easier.

### 4. The Virtual DOM
React keeps a lightweight copy of the DOM in memory. When state changes, it compares the new virtual DOM to the old one and only updates the parts that changed. This is fast.

## What is JSX?

JSX is a syntax extension that lets you write HTML-like code inside JavaScript:

```jsx
// This looks like HTML but it's actually JavaScript
const element = (
  <div className="card">
    <h2>Hello!</h2>
    <p>This is JSX</p>
  </div>
);

// Babel compiles it to this (you never write this yourself)
const element = React.createElement(
  'div',
  { className: 'card' },
  React.createElement('h2', null, 'Hello!'),
  React.createElement('p', null, 'This is JSX')
);
```

JSX differences from HTML:
- `class` → `className`
- `for` → `htmlFor`
- Self-closing tags need a slash: `<img />`, `<input />`
- JavaScript expressions go in `{curly braces}`

## Your first React component

```tsx
// A simple functional component
function Greeting({ name }: { name: string }) {
  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>Welcome to React.</p>
    </div>
  );
}

// Usage
<Greeting name="Alice" />
```
